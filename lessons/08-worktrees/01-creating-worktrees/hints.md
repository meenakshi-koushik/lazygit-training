## Hint 1

You do not need to stash or commit your WIP changes. Git worktrees let you check out another branch in a completely separate directory. Look for the worktree keybinding -- it is not a numbered panel.

## Hint 2

Press `w` to open the worktree options/view. From there, press `n` to create a new worktree. Lazygit will prompt you for a path and a branch. The path should be `../01-creating-worktrees-dashboard` (a sibling directory next to your current sandbox repo).

## Hint 3

When lazygit prompts for the branch, select `feature/dashboard`. After the worktree is created, you should see it listed when you press `w`. You do not need to switch to it -- just creating it is enough. Make sure you stay on `feature/auth` in the original worktree and your uncommitted changes are still visible in the Files panel (`2`).
